Astro Navfolio é um starter de código aberto preparado para sites pessoais. Não é um currículo tradicional de uma única página, nem um template de blog que apenas exibe uma lista de artigos, mas sim organiza apresentação pessoal, entradas comuns, arquivo de artigos, documentação de projetos e componentes de status leve em um único site estático.

Repositório do projeto: [github.com/dodolalorc/astro-navfolio](https://github.com/dodolalorc/astro-navfolio)

## Por que fazer este projeto

Muitos sites pessoais são divididos em múltiplas entradas entre portfólio, blog, currículo, link do GitHub e atividades recentes. O objetivo do Navfolio é colocar esses conteúdos juntos com um custo de manutenção menor: a homepage responsável pela navegação rápida, o blog pela escrita contínua, a área de Projetos pela documentação de projeto e detalhes técnicos.

Este projeto enfatiza conteúdo em primeiro lugar e manutenibilidade a longo prazo. Visualmente, é mais parecido com um notebook de desenvolvedor tranquilo do que com uma página de vendas; em termos de interação, mantém-se leve, deixando efeitos e componentes não roubarem a atenção de leitura.

## Capacidades principais

- Construir sites estáticos usando Astro, adequado para implantação no GitHub Pages, Vercel ou qualquer plataforma de hospedagem estática.
- Gerenciar blog, página About e documentação de projetos usando Markdown / MDX.
- Informações da homepage e site concentradas em `src/config/site.toml`, facilitando substituição por dados pessoais, entradas de navegação e formas de contato.
- Páginas de artigos suportam metadados de título, tempo de leitura, tags, índice e conteúdo relacionado.
- A área de Projetos suporta página de entrada de projeto e documentação de projeto individual, adequada para registrar descrição de repositório, decisões de design e detalhes de implementação.
- Uso de `lucide-astro` fornece um sistema de ícones unificado, reduzindo ícones escritos à mão e desvio de estilo.
- RSS, Sitemap e configuração de implantação do GitHub Pages integrados.

## Estrutura técnica

A implementação do Navfolio mantém o Astro em primeiro lugar:

\`\`\`bash
src/
  components/        Componentes de página, cartões, ferramentas de blog e widgets da homepage
  content/           Conteúdo Markdown / MDX de blog, about e projetos
  config/site.toml   Dados da homepage e configuração de navegação
  data/site.ts       TOML helper de leitura de configuração
  layouts/           Layouts base e layouts de artigos
  pages/             Rotas Astro
  styles/global.css  Variáveis visuais globais e regras de tipografia
\`\`\`

O benefício dessa estrutura é clareza, direto ao ponto, e fácil colaboração com IA: conteúdo, layout e restrições visuais têm limites claros, adicionar nova página não requer introduzir gerenciamento de estado complexo ou framework cliente.

## Direção de design

A linguagem visual do Navfolio é construída ao redor de algumas palavras-chave: leve, tranquilo, estruturado, adequado para leitura. Cartões usam bordas fracas próximas ao GitHub e sombras baixas, efeitos são controlados a deslocamentos muito pequenos e mudanças de transparência, e a página de artigo coloca o corpo principal como centro visual.

A página de Projetos também segue o mesmo princípio. Documentação de projeto não é forçada em cartões pesados de portfólio, mas apresentada como artigos: primeiro explicar o contexto, depois registrar estrutura, capacidades e trade-offs.

## Adequado para uso em quais cenários

Este starter é adequado para os seguintes cenários:

- Quer uma homepage pessoal leve, mantendo a capacidade de escrita de blog.
- Espera colocar projetos de código aberto, documentação de projeto e apresentação pessoal na mesma entrada.
- Precisa de um jardim digital estático sustentável a longo prazo.
- Quer construir um sistema de publicação simples mas organizado usando Astro, MDX e coleções de conteúdo.

## O que pode ser estendido no futuro

O projeto atual já possui o esqueleto básico de um sistema de publicação pessoal. Pode ser continuado:

- Adicionar metadados de repositório, capturas de tela e tags de status para entradas de Projetos.
- Adicionar índice de tags para blog e páginas de tema agregadas por assunto.
- Integrar busca, permitindo artigos e documentação de projeto para pesquisa unificada.
- Adicionar fluxo de atividade pessoal mais real ou cartões de dados do GitHub à homepage.
- Sedimented especificações visuais em mais documentos `.agents/skills`, facilitando iteração contínua.

O foco do Navfolio não é fazer um sistema complexo de uma vez, mas fornecer um ponto de partida suficientemente claro e leve, permitindo que o conteúdo pessoal cresça de forma estável.
